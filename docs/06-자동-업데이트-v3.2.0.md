# 자동 업데이트 — v3.2.0

> 작성일: 2026-07-18
> 새 버전이 나오면 앱이 **시작 시 자동으로 확인 → 팝업 → "지금 업데이트" 클릭 → 다운로드·설치·재시작**.
> Tauri v2 `tauri-plugin-updater` 기반. 관련: [`PROGRESS.md`](../PROGRESS.md) 단계 13.

---

## 동작 방식

```
앱 시작 → check() → GitHub Release "latest" 의 latest.json 조회
        → 새 버전이면 팝업(현재/새 버전·릴리스 노트)
        → "지금 업데이트" → 설치본(.exe) 다운로드(진행률) → 서명 검증 → 설치 → relaunch()
```

- **엔드포인트**: `https://github.com/pyjhoop/study-log/releases/latest/download/latest.json`
  (`tauri.conf.json` `plugins.updater.endpoints`). CI가 릴리스마다 `latest.json`을 생성·첨부한다.
- **서명 검증**: 업데이터는 **서명된 업데이트만** 설치한다. `plugins.updater.pubkey`(공개키)로 검증하고,
  개인키는 GitHub Actions 시크릿(`TAURI_SIGNING_PRIVATE_KEY`)으로 CI 빌드 때 서명에 쓴다.
  → 위·변조된 업데이트는 설치되지 않는다. (이 서명은 **Authenticode 코드 서명과 별개**로,
  SmartScreen "알 수 없는 게시자" 경고는 여전히 존재한다.)
- **설치 모드**: Windows NSIS `passive`(진행 표시만, 상호작용 없이 조용히 설치).

## 확인 시점 (UX)

- **시작 시 자동**: 메인 창의 `UpdateDialog`가 마운트 시 1회 조용히 확인. 있으면 팝업, 없으면 아무 표시 없음.
- **수동**: 설정 › 일반 › "업데이트 확인" 버튼(`check-update` 이벤트 → 메인 창이 확인). 최신이면 "이미 최신 버전입니다" toast.

## ⚠️ 중요한 전제

1. **업데이터가 포함된 버전부터** 자동 업데이트가 된다. v3.1.1 이하(업데이터 없음)는 스스로 못 올라간다.
   → **v3.2.0을 한 번 수동 설치**하면, 그 이후(v3.2.1, v3.3.0 …)부터 자동 팝업·업데이트된다.
2. **서명 개인키를 잃으면** 이후 업데이트에 서명할 수 없어 자동 업데이트가 끊긴다.
   - 키는 GitHub Actions 시크릿 `TAURI_SIGNING_PRIVATE_KEY`(+ 빈 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`)에 등록돼 있다.
   - 공개키(`tauri.conf.json`)와 개인키는 **한 쌍**이라, 공개키를 바꾸면 기존 설치본이 새 업데이트를 못 받는다.
   - 개인키 사본은 안전한 곳(비밀번호 관리자 등)에 백업 권장.

## 릴리스 방법 (변경 없음)

```bash
# 버전 올리고(4곳: tauri.conf.json·Cargo.toml·package.json·사이드바) 커밋 후
git tag v3.2.1 && git push origin v3.2.1
```
→ Actions가 Windows에서 빌드·서명하고 Release에 `*-setup.exe` + `latest.json`(+`.sig`)를 첨부.
설치된 앱들이 다음 시작 때 이 `latest.json`을 보고 업데이트 팝업을 띄운다.

## 검증 안내 (런타임 E2E)

1. v3.2.0을 설치.
2. 코드에서 버전을 올려(예: 3.2.1) `git tag v3.2.1 && git push origin v3.2.1` → Actions 릴리스 완료 대기.
3. 설치된 v3.2.0 앱 재시작 → **업데이트 팝업**(v3.2.1) → "지금 업데이트" → 진행률 → 자동 재시작 → 버전 3.2.1 확인.
4. 최신 상태에서 설정 › "업데이트 확인" → "이미 최신 버전입니다" toast.
