## 🔴 데이터 유실

### 1. GitHub 복원이 부분 실패 시 전체 데이터 소실
- **문제**: `importData`가 `DELETE FROM sessions/subjects` 후 한 행씩 INSERT하는데 트랜잭션이 없어,
  손상/손편집된 백업이 중간에 던지면 **삭제만 되고 일부만 복원**된다("되돌릴 수 없습니다" 안내 직후 유실).
- **수정**(`src/lib/backup.ts`): 파괴적 삭제 **전에** 페이로드를 전량 검증(`validatePayload`) —
  지원 버전(`SUPPORTED_VERSIONS`) + 과목/세션 각 행의 필드 타입 + `settings` 객체 형태.
  문제가 있으면 DB를 건드리기 전에 던진다 → INSERT 도중 실패 지점을 제거.
- **한계**: `tauri-plugin-sql`은 커넥션 풀이라 JS에서 `BEGIN/COMMIT`을 나눠 걸어도 한 트랜잭션이
  보장되지 않는다. 그래서 **원자성(트랜잭션) 대신 사전 검증**으로 실패 가능성을 없애는 전략.
  완전한 원자적 복원이 필요하면 Rust `#[tauri::command]`에서 단일 `sqlx` 트랜잭션으로 처리해야 한다(후속 과제).

### 2. 뽀모도로 켠 채 오버레이 리로드 → 진행 중 세션 강제 일시정지
- **문제**(`src/hooks/usePomodoro.ts`): 세션 리셋 시 `blockStartStudy`를 `0`으로 두는데, 측정 중
  웹뷰가 리로드/재생성되면 `elapsedSec`이 크게 복구돼 다음 틱에 `blockElapsed = elapsed - 0 ≥ 목표`가
  즉시 참 → 공부 중인 세션을 강제 휴식(pause)시키고 사이클 수도 틀어졌다.
- **수정**: `blockStartStudy: args.elapsedSec`로 초기화(새 세션은 ≈0이라 동일, 리로드는 현재 경과를
  블록 시작점으로 잡아 오작동 방지). 자동 pause/resume 실패도 조용히 삼키지 않고 `console.warn`.

### 3. 종료 요약(session-finished) 유실 시 세션 미저장
- **문제**(`src-tauri/src/commands.rs`): `stop_session`은 요약을 emit만 하고 저장은 메인 창 JS 리스너에
  의존. `--autostart` 초기 로드처럼 리스너 등록 전에 시작→종료가 일어나면 세션이 조용히 사라진다.
- **수정**: Rust가 종료 요약을 `PendingFinished`(managed `Mutex<Option<SessionSummary>>`)에 보관하고,
  새 커맨드 `take_pending_finished`로 배수(drain)한다. 메인 창(`useSessionRecorder`)이 **마운트 시** 보관분을
  먼저 저장하고, 이후 이벤트도 처리한다. 이벤트/배수가 겹쳐도 **자연키 dedup**(#5)으로 중복 INSERT 없음.

## 🟠 경합 (동시성)

### 4. 전역 핫키 재등록 경합
- **문제**(`src/hooks/useGlobalShortcuts.ts`): `registerAll`이 직렬화되지 않아 초기 등록과
  `hotkeys-changed` 재등록(또는 연속 저장)이 겹치면 서로의 `register`/`unregisterAll`가 엉켜
  "이미 사용 중" 오탐·누락이 났다.
- **수정**: **generation 토큰**(`genRef`)으로 각 await 뒤 세대 비교 → 나중에 시작한 실행만 살아남는다.

### 5. 크래시 복구/종료 배수 중복 저장
- **문제**: 하트비트 `writeLive`가 정상 종료의 `clearLive` 뒤에 늦게 도착하거나, 저장 직후 죽으면
  다음 실행 복구가 **이미 저장된 세션을 재저장**했다(dedup 가드 없음).
- **수정**(`src/lib/sessions.ts` + `useLiveSessionGuard.ts`): `sessionExists(subject_id, started_at, ended_at)`
  자연키 검사를 추가해 복구 저장·종료 배수 모두 중복이면 skip.

### 6. 단축키 캡처 행 동시 활성
- **문제**(`HotkeyCapture.tsx`): 각 행이 독립 `capturing` 상태 + 자체 `window` keydown 리스너라,
  두 행에서 "변경"을 누르면 키 한 번에 **두 바인딩이 덮어써졌다**.
- **수정**: `HotkeyCapture`를 **제어형**으로 바꾸고 활성 캡처 행(`capturingKey`)을
  `HotkeysSection`이 소유 → 한 번에 하나만 캡처. `stopImmediatePropagation` 안전망 추가.

### 7. 오버레이 설정 stale-closure로 빠른 편집 유실
- **문제**(`OverlaySection.tsx`): `patch`가 렌더 스코프의 `opt`에 병합해, 같은 틱 연속 편집(색+투명도
  드래그) 시 앞 변경이 유실됐다.
- **수정**: 최신 값을 `optRef`로 들고 즉시 갱신해 연속 patch가 서로 쌓이게 함.

## 🟡 견고성

### 8. Alt+F4로 오버레이/피커 창 파괴
- **문제**(`src-tauri/src/lib.rs`): `on_window_event`가 `main`만 close 가드. 테두리 없는 `timer`/`quickstart`
  창은 focus 상태에서 **Alt+F4로 파괴**되면 재시작 전까지 오버레이/시작 핫키가 먹통.
- **수정**: `main`/`timer`/`quickstart` 모두 `CloseRequested` → `prevent_close` + `hide`(파괴 대신 숨김).

### 9. 오버레이가 화면 밖으로 복원(멀티모니터)
- **문제**(`useOverlayWindow.ts`): 저장된 물리 좌표를 무조건 적용해, 그 모니터가 분리/재배치되면
  창이 화면 밖에서 실종.
- **수정**: `clampToMonitors` — `availableMonitors()`로 어떤 모니터와도 안 겹치면 주 모니터 중앙으로.
  권한 `core:window:allow-available-monitors`/`allow-primary-monitor`(`timer.json`). async 리스너 누수도 정리.

### 10. Rust 견고성
- **뮤텍스 poison**(`commands.rs`): 락 헬퍼 `lock()`이 `unwrap_or_else(into_inner)`로 poison을 복구
  (한 번 패닉 후 모든 측정 커맨드가 영구 불능이 되던 것 방지 — 측정 상태는 `Copy` 필드뿐이라 안전).
- **트레이 아이콘 패닉**(`tray.rs`): `expect` → `ok_or(...)?`로 에러 반환(시작 크래시 방지).

### 11. 백업 복원 검증
- 위 #1의 `validatePayload`가 버전·행 스키마까지 검증. `settings`가 객체가 아니면 거부.

## 🟢 통계 표시 오차

### 12. "같은 시점" 비교의 DST·월말 처리 (`stats.ts`)
- **DST**: `prevEnd`를 epoch 초 delta 대신 **달력 연산**(`samePointInPrev`)으로 계산 —
  요일/일자 + 시:분:초를 직접 옮겨 DST 경계 1시간 오차 제거(KST는 무해하나 해외 배포 대비).
- **월말**: 직전 달 같은 일자가 없으면(예: 30일 vs 2월) 그 달 **말일로 clamp** → "전월 전체"로
  뭉개지던 것을 완화(더 공정한 페이스 비교).

### 13. 히트맵 (`stats.ts` / `ContributionHeatmap.tsx`)
- **월 라벨**: 월요일 기준이라 한 칸 밀리던 것을, **그 달 1일이 포함된 열**에 붙이도록(GitHub 규약).
- **"최근 1년" 카운트**: 격자는 53주(≈371일)라 1년보다 길어, **최근 365일 안의 학습일만** 센다.

### 14. 기간 표 증감 경계 (`stats.ts` / `PeriodTable.tsx`)
- **문제**: 창의 가장 오래된 행이 배열 이웃이 없어 증감이 "—"로 끊겼다.
- **수정**: `StatBucket.prevTotal`(직전 기간 합계, 창 밖이어도 totals 맵에서 조회)로 계산 →
  창 경계에서 안 끊김. 직전 기간 기록이 0이면 증감 생략.

## 검증 안내 (런타임 E2E — 사용자 테스트)

`npm run tauri dev`(또는 설치본) 후:
- **복원**: 손상된 백업(필드 누락 등)으로 복원 시도 → toast 에러 + **데이터 그대로**(삭제 안 됨).
- **뽀모도로**: 측정 중 오버레이 F5 리로드 → 즉시 휴식으로 안 넘어가고 현재 집중 이어감.
- **종료 유실**: (설치본) 자동 시작 직후 빠르게 시작→종료해도 세션 저장됨.
- **핫키**: 설정에서 조합 연속 저장/변경 → 오탐·누락 없이 최신 조합만 동작. 두 행 동시 "변경" 방지.
- **오버레이**: 다른 모니터 뽑고 재실행 → 창이 주 모니터 중앙으로 복귀.
- **Alt+F4**: 오버레이/피커에 포커스 두고 Alt+F4 → 숨김(파괴 아님), 이후에도 정상 표시/시작.
- **통계**: 히트맵 월 라벨 정렬, 표 증감이 오래된 행에서도 표시.
