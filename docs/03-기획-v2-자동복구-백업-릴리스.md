# v2 기능 기획서 (Feature Spec — V2)

> 작성일: 2026-07-18
> 상태: **V2 기획 확정**
> 선행 문서: [`01-기술스택-결정.md`](./01-기술스택-결정.md) · [`02-기획-v1-핵심기능.md`](./02-기획-v1-핵심기능.md)

V1(단계 0~8, 버전 1.0.0)이 완료된 뒤 추가하는 **v2 기능 4종**의 세부 기획서다.
각 기능은 독립적으로 구현·검증하며, 순서대로 진행한다.

| # | 기능 | 한 줄 요약 | Rust 변경 |
|---|------|-----------|-----------|
| F1 | **비정상 종료 세션 자동 복구** | 진행 중 세션을 주기적으로 DB에 기록해 크래시/강제종료 시 마지막 시점까지 자동 저장 | 없음 |
| F2 | **Windows 자동 시작** | 부팅 시 앱이 (조용히 트레이에) 자동 상주 | 소량 |
| F3 | **GitHub 백업 / 복원** | 전체 데이터를 GitHub 레포에 JSON으로 백업하고 새 PC에서 복원 | 없음(플러그인만) |
| F4 | **GitHub Actions 릴리스 자동화** | 버전 태그 push → 클라우드에서 빌드 → Release 자동 생성 | 없음 |

버전은 이 v2 완료 시 **2.0.0**으로 승격한다(`tauri.conf.json`·`Cargo.toml`·`package.json`·사이드바).

세 기능(F1~F3)은 모두 **기존 `settings` 테이블/스키마로 충분** → DB 마이그레이션 없음.

---

## 배경 — 왜 지금 이 기능들인가

1. **데이터 유실 위험이 두 방향으로 존재한다.**
   - *진행 중 세션*: 측정 상태는 Rust 인메모리(`Mutex<Measurement>`)에만 있고 `stop_session` 시점에만 DB INSERT된다. 트레이 상주(창 X → 숨김) 구조라 사용자가 "종료"를 잘 누르지 않으므로, **타이머를 켠 채 PC를 끄거나 앱이 강제 종료되면 그 세션 전체가 사라진다.** → **F1**
   - *전체 DB*: 모든 데이터(과목·세션·설정)가 로컬 단일 SQLite(`%APPDATA%\com.studylog.app\studylog.db`)에만 있다. PC 고장·포맷·재설치 시 전부 유실. → **F3**
2. **매일 쓰는 상주형 앱**이므로 부팅 때 수동 실행이 번거롭다. → **F2**
3. **미서명 NSIS 빌드를 수동으로** 만들어 릴리스에 올리고 있다. 배포를 재현 가능하게 자동화한다. → **F4**

## F1 — 비정상 종료 세션 자동 복구

### 요구사항
- `[V2]` 측정(Running/Paused) 중 앱/PC가 비정상 종료돼도, **다음 실행 시 그 세션이 마지막으로 살아있던 시점까지 자동 저장**된다(사용자에게 묻지 않음).
- `[V2]` PC가 꺼져 있던 시간은 공부 시간에 포함하지 않는다(마지막 기록 시점 = 종료 시각).
- `[V2]` 정상 종료(stop) 후 재실행 시에는 아무 일도 없어야 한다(중복 저장 금지).
- `[V2]` 웹뷰만 리로드된 경우(Rust 프로세스 생존)에는 복구를 시도하지 않는다.

### 설계
DB 접근을 JS로 통일하는 기존 규약에 맞춰 **프론트(메인 창)에서만** 구현한다. Rust/스키마 변경 없음.

**저장 계층** — `src/lib/liveSession.ts`, `settings` 테이블 key `"live_session"`
- 저장 객체: `{ subject_id, started_at, ended_at, duration_sec, paused_sec }`
  - `ended_at` = 마지막 하트비트 시각, `duration_sec`/`paused_sec` = 그 시점의 파생값
  - elapsed 계산은 `lib/time.ts`의 기존 공식(`computeElapsed`)을 그대로 재사용
- API: `writeLive(v)` / `readLive()` / `clearLive()`

**감지·갱신 훅** — `src/hooks/useLiveSessionGuard.ts` (메인 창에서 1회 마운트)
1. **복구(마운트 시 1회, 최우선)**: `get_session_state`가 `idle`인데 `readLive()`에 잔재가 있으면 = 비정상 종료로 판단.
   `duration_sec > 0`이면 그 값으로 세션 INSERT(기존 `saveSession` 재사용) → `clearLive()` → `session-saved` emit → toast(`비정상 종료된 세션 자동 저장: 과목 · N분`).
   Rust가 running/paused면(웹뷰만 리로드) 복구하지 않는다.
2. **지속화**: `session-changed` 구독 → running/paused면 `writeLive()`, idle이면 `clearLive()`.
3. **하트비트**: running/paused 동안 **20~30초 간격**으로 `writeLive()`. 크래시 시 손실 ≤ 하트비트 간격. 창이 트레이로 숨겨져도 webview 타이머는 동작(숨김 시 ~1분 granularity — 복구엔 충분).

### 트레이 "종료" 케이스
트레이 "종료"는 `app.exit(0)`이라 running 세션에 `stop_session`을 거치지 않는다 → 위 자동 복구가 다음 실행 때 마지막 하트비트까지 저장(최대 하트비트 간격만큼 과소 계상, 허용). (선택 개선: 트레이 종료 시 running이면 먼저 정상 종료 — 이번 범위 밖.)

## F2 — Windows 자동 시작 (부팅 시 트레이 상주)

### 요구사항
- `[V2]` 설정에서 "Windows 시작 시 자동 실행"을 켜고 끌 수 있다.
- `[V2]` 켜져 있으면 부팅 시 앱이 자동 실행되되 **창을 띄우지 않고 트레이에만 조용히 상주**한다(핫키·측정 대기).
- `[V2]` 트레이 좌클릭으로 창을 열 수 있다(기존 동작).

### 설계
이미 트레이 상주(X→hide, 완전 종료는 트레이 "종료"만) 구조라 자동 시작과 궁합이 좋다.

**플러그인**
- Cargo: `tauri-plugin-autostart = "2"` · npm: `@tauri-apps/plugin-autostart`
- `lib.rs`의 `#[cfg(desktop)]` 블록에 등록:
  `tauri_plugin_autostart::init(MacosLauncher::LaunchAgent, Some(vec!["--autostart"]))`
  (Windows는 레지스트리 `Run` 키 방식 — NSIS와 무관하게 동작. `--autostart` 인자로 부팅 실행을 식별.)

**조용한 상주**
- `tauri.conf.json`의 `main` 창을 `visible: false`로 변경.
- `lib.rs` setup에서 `std::env::args()`에 `--autostart`가 **없으면** main 창 `.show()`(정상 실행), 있으면 숨긴 채 트레이만.
- main이 처음부터 hidden이어도 webview는 살아있어 저장 리스너·핫키·백업·F1 가드가 모두 정상 동작한다.

**UI/권한**
- `GeneralSection.tsx`에 `ui/Switch` 토글 추가. 마운트 시 `isEnabled()`로 초기값, 토글 시 `enable()`/`disable()`. 에러는 toast.
- `capabilities/main.json`에 `autostart:default` 추가.

## F3 — GitHub 백업 / 복원 (GitHub API + JSON)

### 요구사항
- `[V2]` 설정에서 GitHub 저장소(owner/repo/branch/path)와 **Personal Access Token(PAT)** 을 등록한다.
- `[V2]` "지금 백업" 버튼 → 전체 데이터를 **JSON 한 파일**로 레포에 커밋(생성/갱신).
- `[V2]` "GitHub에서 복원" 버튼 → 레포의 JSON을 받아 로컬 DB를 그 시점으로 복원(확인 모달 후).
- `[V2]` 토큰/레포 미설정이면 백업·복원 버튼 비활성. 모든 오류는 toast.

### 설계
CORS 없이 GitHub REST API를 호출하기 위해 **`tauri-plugin-http`** 를 쓴다. 백업/복원 로직은 전부 JS.

**플러그인/권한**
- Cargo: `tauri-plugin-http = "2"` · npm: `@tauri-apps/plugin-http` · `lib.rs`에 `.plugin(tauri_plugin_http::init())`
- `capabilities/main.json`에 GitHub 스코프 http 권한:
  `{ "identifier": "http:default", "allow": [{ "url": "https://api.github.com/*" }] }`

**설정 저장** — `settings` 테이블
- `"backup_config"` = `{ owner, repo, branch(기본 "main"), path(기본 "study-log-backup.json") }`
- `"github_token"` = PAT
- ⚠️ **토큰은 로컬 SQLite에 평문 저장**된다(개인용 로컬 앱이라 허용). UI에 안내 문구를 둔다. OS 키체인 저장은 Not-V2.

**데이터 계층** — `src/lib/backup.ts`
- `exportData()`: `subjects`/`sessions`/`settings` 전량 SELECT → `{ version:2, exported_at, subjects[], sessions[], settings{} }`.
  **민감/휘발 key 제외**: settings에서 `github_token`·`backup_config`·`live_session`은 백업에 담지 않는다(토큰 유출·잔여 상태 방지).
- `pushBackup()`:
  1. `GET /repos/{owner}/{repo}/contents/{path}?ref={branch}` → 기존 파일 `sha` 조회(404면 신규)
  2. `PUT /repos/{owner}/{repo}/contents/{path}` body `{ message, content: base64(utf8(json)), branch, sha? }`
  - 헤더: `Authorization: token <PAT>`, `Accept: application/vnd.github+json`
  - 한글(과목명/메모) 때문에 `btoa` 직접 금지 → `TextEncoder`로 UTF-8 바이트 → base64.
  - 성공 시 `settings`의 `last_backup_at` 갱신.
- `restoreBackup()`: 파일 GET → JSON 파싱 → **전체 교체**(개인 복원): `DELETE sessions` → `DELETE subjects` → 백업 id 그대로 INSERT(FK 보존) → settings upsert(민감 key 제외). 파괴적이므로 **`ConfirmDialog`** 뒤에서만.

**UI** — `src/components/settings/BackupSection.tsx` (설정 화면 카드로 연결)
- GitHub 설정 입력 + 저장, "지금 백업"(마지막 백업 시각 표시), "GitHub에서 복원"(확인 모달 → 성공 시 `session-saved` emit로 대시보드 새로고침).

### 백업 데이터 형식(JSON) 예시
```json
{
  "version": 2,
  "exported_at": 1752800000,
  "subjects": [{ "id": 1, "name": "수학", "color": "#6366f1", "sort_order": 0, "archived": 0, "created_at": 1752000000 }],
  "sessions": [{ "id": 12, "subject_id": 1, "started_at": 1752790000, "ended_at": 1752793600, "duration_sec": 3600, "paused_sec": 0, "memo": null, "created_at": 1752793600 }],
  "settings": { "daily_goal_min": 120, "hotkeys": { "...": "..." } }
}
```

## F4 — GitHub Actions 빌드·릴리스 자동화

### 요구사항
- `[V2]` 버전 태그(`v*`)를 push하면 클라우드에서 **Windows 빌드**가 돌고, 산출물(NSIS 설치파일)이 **GitHub Release**로 자동 첨부된다.
- `[V2]` 수동 실행(`workflow_dispatch`)도 가능.

### 설계 — `.github/workflows/release.yml`
- **트리거**: `push` `tags: ['v*']` + `workflow_dispatch`.
- **러너**: `windows-latest` (앱이 Windows 전용, NSIS 타깃).
- **스텝**:
  1. `actions/checkout`
  2. `actions/setup-node`(Node 20+) + npm 캐시 → `npm ci`
  3. `dtolnay/rust-toolchain@stable` + `swatinem/rust-cache`(Rust 빌드 캐시)
  4. `tauri-apps/tauri-action` — 빌드 + Release 생성/업로드를 한 번에.
     - `tagName`/`releaseName`을 태그 기준으로, `releaseDraft: false`.
     - `GITHUB_TOKEN`(기본 제공)으로 Release 업로드.
- **주의점**:
  - CI 러너는 **Smart App Control이 없어** 로컬에서 겪은 빌드 차단(`os error 4551`) 문제가 없다 → 클린 빌드에 유리.
  - **미서명 빌드** 유지(코드 서명은 Not-V2). 서명 붙일 자리(secrets)는 주석으로 남긴다.
  - `productName`이 한글("학습기록")이라 **WiX(MSI) 비활성 · NSIS만**(로컬과 동일, `bundle.targets: ["nsis"]`).

### 릴리스 흐름(사용 예)
```bash
# 버전 올리고(2.0.0) 커밋 후
git tag v2.0.0
git push origin v2.0.0    # → Actions가 빌드 후 Release v2.0.0에 설치파일 첨부
```

## 구현·검증 순서

1. **F1** (자립적, Rust 무변경) → 2. **F2** → 3. **F3** → 4. **버전 2.0.0 승격 + 로컬 빌드 검증** → 5. **F4**(워크플로) → 태그 push로 실제 릴리스 확인.

각 단계: `npm run build`(tsc+vite) + 필요 시 `cargo check` 선검증 → 사용자 E2E → `PROGRESS.md` 갱신 → 커밋.

## 범위 밖 (Not V2)
- 코드 서명(SmartScreen 경고 제거).
- 자동/주기적 백업, 백업 충돌 병합(수동 백업/복원만).
- OS 키체인 토큰 저장(현재 로컬 SQLite 평문).
- macOS/Linux 빌드(Windows 전용 앱).
