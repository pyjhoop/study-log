# 핵심 기능 기획서 (Feature Spec)

> 작성일: 2026-07-18
> 상태: **V1 기획 확정**
> 선행 문서: [`01-기술스택-결정.md`](./01-기술스택-결정.md)

이 문서는 학습기록 앱의 **핵심 기능(핫키 · 타이머 오버레이 · 측정/기록 · 부가기능)** 을 하나로 묶은 세부 기획서다. 각 요구사항에는 **`[V1]`** 태그를 붙였고, 이번 버전 대상이 아닌 것은 맨 끝 [12. 향후(Not V1)](#12-향후-not-v1)로 분리했다.

## 1. 개요 & 사용 시나리오

**핵심 시나리오:** 사용자는 인강·PDF·문제집 앱 등 **다른 앱으로 공부하면서** 그 시간을 측정한다. 따라서 study-log 창을 보지 않고도 측정을 제어할 수 있어야 하고, 공부 중인 화면 위에 경과 시간이 항상 보여야 한다.

이 시나리오에서 두 기능이 앱의 정체성을 만든다:

1. **전역 핫키** — 다른 앱을 쓰는 도중에도 측정을 시작/종료/일시정지할 수 있다.
2. **항상-위 플로팅 타이머** — 공부 중인 앱 위에 작은 타이머 창이 떠서 경과 시간을 보여준다.

여기에 통계 대시보드, 뽀모도로/목표시간, 트레이 상주를 더해 "완성된 앱"으로 만든다.

## 4. 창(Window) 구조

같은 프론트엔드 번들을 3개의 창이 공유하고, `getCurrentWindow().label`로 어떤 화면을 렌더할지 분기한다. 오버레이/피커는 `visible:false`로 시작하고 필요할 때 `show()`.

| 창(label) | 역할 | 주요 옵션 |
|-----------|------|-----------|
| `main` | 4개 화면(대시보드 등) | 일반 창, resizable |
| `timer` | 공부 중 경과 시간 표시 | `decorations:false`, `transparent:true`, `alwaysOnTop:true`, `resizable:true`, `skipTaskbar:true`, `visible:false`, `minWidth`/`minHeight` |
| `quickstart` | 시작 시 과목 선택 팝업 | `decorations:false`, `alwaysOnTop:true`, `center`, `focus:true`, `skipTaskbar:true`, `visible:false` |

- `timer`는 `transparent:true` + CSS 반투명 배경(예: `bg-black/60 rounded-xl`)으로 알약 모양 오버레이를 만든다.

## 5. 측정 흐름 & 상태 엔진 `[V1: R1]`

### 5-1. 상태
측정 상태는 **여러 창이 함께 봐야 하므로 Rust가 단일 소스**로 관리하고, 변경 시 이벤트로 브로드캐스트한다.

```
enum Status { Idle, Running, Paused }
struct MeasurementState {
  status: Status,
  subject_id: Option<i64>,
  started_at: Option<i64>,        // epoch sec
  accumulated_paused_sec: i64,    // 누적 일시정지 시간
  paused_at: Option<i64>,         // 현재 정지 시작 시각(정지 중일 때)
}
```

### 5-2. 커맨드 (프론트 `invoke`)
- `start_session(subject_id)` — Idle일 때만 Running으로.
- `pause_session()` / `resume_session()` — Running ↔ Paused.
- `stop_session()` — 요약(공부 시간 등) 산출 후 Idle로. **세션 저장은 메인 창 JS가 sql 플러그인으로 수행**(DB 접근을 JS 한 곳으로 통일).
- `get_session_state()` — 창이 새로 뜨거나 리로드될 때 현재 상태 동기화.

변경 시 `emit_all("session-changed", state)`로 모든 창에 알린다.

### 5-3. 경과 시간 계산
```
elapsed = now - started_at - accumulated_paused_sec - (paused ? now - paused_at : 0)
```
오버레이가 `setInterval(1s)`로 이 값을 다시 계산해 그린다 → 초당 이벤트 전송 없이 드리프트 없는 표시. 상태 변화는 `session-changed` 구독으로 반영.

## 6. 전역 핫키 `[V1: R2]`

`tauri-plugin-global-shortcut` 사용. 앱 시작 시 `settings`에서 바인딩을 읽어 `register()`.

| 기능 | 기본 단축키 | 동작 |
|------|------------|------|
| 측정 시작 | `Ctrl+Alt+S` | `quickstart` 창 표시(과목 선택). Idle일 때만 |
| 측정 종료 | `Ctrl+Alt+E` | `stop_session` → 저장 |
| 일시정지/재개 | `Ctrl+Alt+P` | Running ↔ Paused 토글 |
| 대시보드 열기 | `Ctrl+Alt+D` | `main` 창 표시 + 포커스 |
| 오버레이 숨김/표시 | `Ctrl+Alt+H` | `timer` 창 show/hide 토글 |

- 설정 화면에서 각 바인딩을 **커스터마이즈**(키 캡처 UI). 저장 시 `unregisterAll()` 후 재등록.
- 이미 다른 앱이 점유한 조합은 등록 실패 → "이 조합은 사용 중" 안내.
- 상태에 맞지 않는 입력은 무시(Running 중 시작키, Idle 중 종료키 등).

## 7. 빠른 시작 피커 `[V1: R3]`

시작 시 **과목을 먼저 정하는** 흐름을, 전역 핫키에서도 매끄럽게 처리하기 위한 작은 팝업.

1. `Ctrl+Alt+S` → `quickstart` 창 `show()` + `setFocus()` (다른 앱 위 중앙에 뜸).
2. 과목 목록 표시. **타이핑으로 필터**, `↑/↓` + `Enter` 또는 **숫자키(1~9)** 로 즉시 선택, `Esc`로 취소.
3. 선택 → `start_session(subject_id)` → `quickstart` 숨김 → `timer` 오버레이 자동 표시.
4. 과목이 하나도 없으면 "과목 먼저 만들기" 안내 + 과목 관리 화면으로 유도.

## 8. 타이머 오버레이 `[V1: R4]`

### 8-1. 표시
경과 시간 `HH:MM:SS` 기본. 설정에 따라 과목명 · 색 점 · 일시정지 배지 · 목표 대비 % 추가. 정지 중이면 색 변경/깜빡임으로 구분.

### 8-2. 상호작용 (테두리 없는 창이라 커스텀 처리)
- **이동**: 드래그 영역에 `data-tauri-drag-region` 부여(또는 `getCurrentWindow().startDragging()`).
- **크기 조절**: 우하단 그립에서 `startResizeDragging('SouthEast')`. `minWidth`/`minHeight`로 하한.
- **글자 크기**: 기본은 창 크기에 비례(설정으로 고정도 가능).
- **숨김/표시**: 핫키(`Ctrl+Alt+H`) 또는 트레이 메뉴. 마지막 위치·크기는 `settings`에 저장 후 복원.
- **미니 컨트롤**: 마우스 오버 시에만 일시정지/종료 버튼 노출 → 평소엔 시간만 깔끔하게.
- 측정 시작 시 자동 `show()`, 종료 시 자동 `hide()`(설정으로 유지 가능).

## 9. 데이터 모델 (SQLite)

시간은 **Unix epoch(초, INTEGER)** 로 저장하고, 집계 시 `localtime`으로 변환해 로컬 날짜 기준으로 그룹핑한다.

```sql
-- 과목
CREATE TABLE subjects (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  name       TEXT NOT NULL,
  color      TEXT NOT NULL DEFAULT '#6366f1',
  sort_order INTEGER NOT NULL DEFAULT 0,
  archived   INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL
);

-- 학습 세션 (측정 1회 = 1행)
CREATE TABLE sessions (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  subject_id   INTEGER NOT NULL REFERENCES subjects(id),
  started_at   INTEGER NOT NULL,     -- epoch sec
  ended_at     INTEGER NOT NULL,     -- epoch sec
  duration_sec INTEGER NOT NULL,     -- 실제 공부 시간(일시정지 제외)
  paused_sec   INTEGER NOT NULL DEFAULT 0,
  memo         TEXT,
  created_at   INTEGER NOT NULL
);
CREATE INDEX idx_sessions_started ON sessions(started_at);
CREATE INDEX idx_sessions_subject ON sessions(subject_id);

-- 앱 설정(키-값): 핫키 바인딩, 오버레이 옵션, 뽀모도로 설정, 목표시간 등 (JSON 문자열)
CREATE TABLE settings (
  key   TEXT PRIMARY KEY,
  value TEXT NOT NULL
);
```

**집계 쿼리 예시**
```sql
-- 일간
SELECT date(started_at,'unixepoch','localtime') AS d, SUM(duration_sec)
FROM sessions GROUP BY d;
-- 주간 / 월간
strftime('%Y-%W', started_at,'unixepoch','localtime')   -- 주
strftime('%Y-%m', started_at,'unixepoch','localtime')   -- 월
```

## 10. 부가기능

### 10-1. 뽀모도로 / 목표시간 `[V1: R5]`
- **뽀모도로 모드**(설정 토글): 집중(기본 25분)/휴식(5분)/롱브레이크 주기. 켜면 오버레이가 **남은 시간 카운트다운** + 현재 사이클을 표시하고, 사이클 종료 시 알림 후 자동 전환한다. 집중 구간은 `sessions`에 공부 시간으로 적립.
- **일일 목표시간**: `settings`에 오늘 목표(분)를 저장. 대시보드에 **진행 링/바**(오늘 누적 ÷ 목표)로 표시. 오버레이 옵션으로 "목표 대비 %"도 표시 가능.

### 10-2. 시스템 트레이 `[V1: R6]`
- 좌클릭: 메인 창 표시/포커스. 우클릭 메뉴: `측정 시작/종료`, `일시정지/재개`, `오버레이 표시/숨김`, `대시보드 열기`, `종료`.
- **메인 창 닫기(X) → 종료 대신 트레이로 숨김**(백그라운드 유지, 핫키 계속 동작). 트레이 "종료"로만 완전 종료.
- 측정 중이면 트레이 툴팁에 경과 시간·과목 표시.

### 10-3. 오버레이 커스터마이즈 `[V1: R7]`
설정 화면에서: 투명도(배경 alpha), 글자 크기 모드(창 비례/고정), 글자·배경 색, 표시 항목 토글(경과시간/과목명/색점/목표%/일시정지 배지), 항상 위 on/off. 값은 `settings`(JSON)에 저장하고, 변경 시 `timer` 창에 이벤트로 즉시 반영.

**검증 (end-to-end)**
- **전역 핫키**: 앱을 백그라운드에 두고 **다른 앱 포커스 상태**에서 시작→피커→과목 선택→오버레이 표시→종료 저장까지 동작.
- **오버레이**: 다른 앱(메모장/브라우저) 위에 항상 표시, 드래그 이동·우하단 크기조절·`Ctrl+Alt+H` 숨김/표시, 재실행 시 위치·크기 복원.
- **뽀모도로/목표**: 짧은 주기(집중 1분 등)로 사이클 전환·알림·집중 시간 적립, 대시보드 목표 링 갱신.
- **트레이**: 메인 X → 트레이 숨김(핫키 계속 동작), 메뉴 각 항목, "종료"로만 완전 종료.
- **통계**: 서로 다른 날짜의 세션을 넣고 일/주/월 집계가 로컬 날짜 기준으로 정확한지 확인.
