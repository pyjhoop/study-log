## G1 — 오버레이 레이아웃(variant) 6종 `[V3]`

### 배경
V1의 타이머 오버레이는 **단일 알약(pill) 모양** 하나뿐이었다. "예쁜 UI" 목표에 맞춰, 사용자가 취향/상황에 따라 오버레이 배치를 고를 수 있게 **선택 가능한 레이아웃 6종**으로 확장한다.

### 요구사항
- `[V3]` 오버레이 레이아웃을 6종 중 하나로 선택할 수 있다.
- `[V3]` 색·투명도·글자색/크기·표시항목 토글 등 **기존 오버레이 옵션은 레이아웃과 무관하게 공유**된다(배치/모양만 교체).
- `[V3]` 설정에서 레이아웃을 바꾸면 측정 중인 타이머 창에 **즉시** 반영된다.
- `[V3]` 선택한 레이아웃은 재시작 후에도 복원된다.

### 6종 레이아웃

| variant | 설명 |
|---------|------|
| `pill` | 알약 (기존 기본형) |
| `digital` | 큰 디지털 시계 |
| `minimal` | 배경 없이 시간만 (text-shadow로 가독성 확보) |
| `ring` | 목표 진행 원형 링 + 가운데 시간 |
| `bar` | 하단 목표 진행 막대 |
| `pomodoro` | 사이클 라벨 + 남은 시간 (뽀모도로 꺼지면 경과 시간 폴백) |

### 설계
- `lib/overlaySettings.ts`: `OverlayOptions`에 `variant` 필드(+`OverlayVariantId` 타입) 추가. 기본값 `"pill"`. 기존 spread 병합으로 누락 자동 보정(구버전 설정 호환).
- **리팩터** `windows/TimerOverlay.tsx`: 데이터 계산(시간/서브라인/색/과목/뽀모도로/**목표 진행률**)을 `OverlayViewModel`로 묶어 각 variant에 위임. 드래그 영역·마우스오버 미니 컨트롤(재개/일시정지/종료)·리사이즈 그립·항상 위는 **공용 크롬**으로 래퍼에 유지 → variant는 "그리기"만 담당.
  - 목표% 로드 조건을 `show.goalPct || variant ∈ {ring, bar}`로 확장(링/막대는 진행률이 핵심이라 항상 필요).
- 신규 `components/overlay/`: `types.ts`(뷰모델), `index.tsx`(레지스트리 + `OverlayVariantView` 스위처), `{Pill,Digital,Minimal,Ring,Bar,Pomodoro}Variant.tsx`. 링은 대시보드 `GoalRing`의 SVG 패턴을 viewBox로 축약해 창 크기에 맞춰 스케일.
- **설정**(`components/settings/OverlaySection.tsx`): 상단에 **레이아웃 선택 버튼 6개**(선택 강조), 미리보기를 **선택한 variant의 실제 렌더**(샘플 뷰모델)로 교체해 WYSIWYG. 색/투명도/표시항목 컨트롤은 그대로. 변경은 기존대로 즉시 저장 + `overlay-options-changed` 이벤트로 타이머 창 실시간 전환.

### 검증 (E2E)
측정 시작 → 오버레이 표시, 설정 "오버레이"에서 레이아웃 6종을 바꿀 때 타이머 창이 **즉시** 전환. 각 레이아웃에서 드래그·리사이즈·마우스오버 컨트롤·색/투명도/표시항목·목표% 정확. 재시작 시 선택 레이아웃 복원.

## G2 — 대시보드 학습 잔디 (히트맵) `[V3]`

### 배경
일/주/월 차트는 있지만, **1년 전체를 한눈에** 보는 뷰가 없었다. GitHub 컨트리뷰션 그래프 형태로 매일의 학습량을 시각화해 꾸준함을 보여준다.

### 요구사항
- `[V3]` 대시보드 최하단에 **최근 약 1년(53주)** 의 일별 학습량 히트맵을 표시한다.
- `[V3]` 색 진하기는 그날 학습량에 비례하고, **일일 목표 대비 비율**을 기준으로 단계를 나눈다.
- `[V3]` 셀에 hover하면 날짜 + 공부시간 툴팁이 뜬다.
- `[V3]` 측정 종료 시 오늘 칸이 자동 반영된다.

### 설계 (순수 프론트, 기존 `useStats.rows` 재사용)
- `lib/stats.ts` `buildHeatmap(rows, weeksCount=53, now)`: 세션을 날짜별 합계로 접고 **최근 53주(≈1년)** 를 **월요일 시작**(앱의 주 경계 규약과 일치) 열들로 만든다. 각 주는 7칸(월~일). 마지막 열이 이번 주이고 오늘 이후 날짜는 `inRange:false`(빈 칸)로 정렬만 유지. 월이 바뀌는 열에 월 라벨(`months`). 타입 `HeatCell`/`Heatmap` 추가.
- `components/dashboard/ContributionHeatmap.tsx`: 열=주 / 행=요일 격자, 셀 11px.
  - **색 단계 0~4는 일일 목표(`goalMin`) 대비 비율**(0 / <50% / <100% / <150% / ≥150%)로 결정. 목표 미설정(0)이면 절대 시간(30분/1시간/2시간)으로 폴백.
  - 색은 앱 테마 **teal** 스케일 + 라이트/다크 대응. 월 라벨(상단)·요일 라벨(월/수/금)·범례(적음→많음)·"최근 1년 N일 학습" 카운트. 셀 `title`에 `formatDayLabel · 공부시간`(hover 툴팁). 좁은 화면 대비 `overflow-x-auto`.
- `components/dashboard/DashboardScreen.tsx`: 최근 세션 카드 아래에 `<ContributionHeatmap rows={rows} goalMin={goalMin} />` 연결.

> ⚠️ 이 히트맵의 초기 구현에는 월 라벨 정렬·"최근 1년" 카운트 관련 표시 오차가 있었고, 직후 [`05-안정성-보강-v3.1.1.md`](./05-안정성-보강-v3.1.1.md) §13에서 수정됐다.

### 검증 (E2E)
대시보드 하단에서: ①기록 있는 날이 teal로, 공부량이 많을수록 진하게 ②오늘 이후 날짜는 빈 칸 ③셀 hover 시 날짜+공부시간 툴팁 ④월/요일 라벨·범례 정렬 ⑤측정 종료 시 오늘 칸 자동 반영(`session-saved`) ⑥라이트/다크 색 대비 ⑦일일 목표를 바꾸면 색 단계 기준 변화.
