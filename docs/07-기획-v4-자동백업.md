# v4 기능 기획서 (Feature Spec — V4)

> 작성일: 2026-07-18
> 상태: **구현 완료** (v4.0.0)
> 선행 문서: [`03-기획-v2-자동복구-백업-릴리스.md`](./03-기획-v2-자동복구-백업-릴리스.md) (F3 GitHub 백업) · [`06-자동-업데이트-v3.2.0.md`](./06-자동-업데이트-v3.2.0.md)

V2에서 만든 GitHub 백업(F3)은 설정 화면 **"지금 백업" 버튼으로만** 동작하는 완전 수동이었다. v4는 **세션을 끝낼 때마다 최신 데이터를 자동으로 백업**하고, 백그라운드로 처리한 뒤 시스템 알림으로 간단히 알린다.

| # | 기능 | 한 줄 요약 | Rust 변경 |
|---|------|-----------|-----------|
| H1 | **세션 종료 시 GitHub 자동 백업** | 정상 종료 저장 직후 백그라운드로 백업 + 성공 시 시스템 알림 | 없음(capability만) |

버전은 이 v4 완료 시 **4.0.0** 으로 승격한다(`tauri.conf.json`·`Cargo.toml`·`package.json`·사이드바).

---

## 배경 — 왜 자동 백업인가

- V2의 백업은 사용자가 설정 화면에 들어가 **"지금 백업"을 눌러야만** 실행됐다(`BackupSection`이 유일한 `pushBackup` 호출부). 실제로는 잘 누르지 않게 되어 백업이 오래 방치되기 쉽다.
- 매 세션 종료는 "데이터가 막 늘어난 시점"이라 **자동 백업 트리거로 가장 자연스럽다.**
- 단, 저장·통계 갱신 등 사용자 경험을 방해하면 안 되므로 **백그라운드(fire-and-forget)** 로 처리하고 결과만 가볍게 알린다.

## H1 — 세션 종료 시 GitHub 자동 백업

### 요구사항
- `[V4]` GitHub 백업이 **설정돼 있으면**, 세션이 정상 종료·저장될 때마다 자동으로 백업을 푸시한다.
- `[V4]` 백업 성공 시 **시스템 알림**("GitHub 백업 완료")으로 알린다.
- `[V4]` 백업은 **백그라운드**로 처리해 세션 저장 toast·통계 갱신을 막지 않는다.
- `[V4]` 미설정이면 아무 동작도, 알림도 없다(조용히 통과).
- `[V4]` 백업 실패(오프라인·토큰 오류 등)는 **조용히 콘솔에만** 남긴다(매 세션 알림 스팸 방지). 세션 저장 자체는 항상 정상.
- `[V4]` 짧은 세션을 연달아 종료해도 백업이 **겹치지 않는다.**

### 설계 (기존 백업 로직 재사용, 새 IPC/플러그인 없음)

**자동 백업 진입점** — `lib/backup.ts` `autoBackup()` 신설
- 기존 `getBackupConfig` / `getToken` / `isConfigReady` / `pushBackup`을 **조합만** 한다.
- 미설정이면 `null` 반환(무동작).
- **`autoBackupInFlight` 가드**로 연속 종료 시 백업 겹침 방지.
- 성공 시 `pushBackup`이 `last_backup_at`도 갱신(수동 백업과 동일 경로).

**트리거** — `hooks/useSessionRecorder.ts`
- 저장이 성공하는 **유일한 메인 창 지점**의 `record()`에서, `emitSessionSaved()` 직후 `void runAutoBackup()`을 **fire-and-forget**으로 붙인다(저장 toast·통계 갱신 비차단).
- 성공 시 시스템 알림, **실패는 조용히 콘솔만**.
- **범위 주의**: 복구 경로(`useLiveSessionGuard`)·수동 복원(`BackupSection`)은 `record()`를 거치지 않아 트리거되지 않는다 → **정상 종료 경로만 자동 백업**(의도된 범위).

**알림 헬퍼** — `lib/notify.ts` 신설
- `usePomodoro`가 쓰던 `@tauri-apps/plugin-notification` 패턴(권한 확인/요청 후 `sendNotification`, 실패 무시)을 재사용 헬퍼로 추출.

**권한** — `capabilities/main.json`
- 메인 창에 notification 4종(`default` / `allow-notify` / `allow-is-permission-granted` / `allow-request-permission`) 추가. 플러그인은 이미 Cargo/lib.rs에 등록돼 있어 **capability만** 부여.

### 설계 결정 — 진행 중 통계 실시간 반영은 현행 유지 (코드 변경 없음)
대시보드 통계 위젯은 `useStats` → `fetchStatRows`로 **저장된 세션만** 쿼리하고 `session-saved`로만 갱신되는 기록성 뷰다. 아직 DB에 없는 진행 중 세션을 끼워 넣으면 의미가 애매해지고(자정 넘김·뽀모도로 일시정지) 엣지케이스가 늘어난다. 진행 중 실시간 피드백은 오버레이의 `LiveMeasure` 초시계가 담당하므로, **저장 완료 시점에만 갱신하는 현재 설계를 유지**한다.

### 검증 (E2E)
`npm run tauri dev` 후:
- **설정된 경우**: 설정 "GitHub 백업"에 owner/repo/token 저장 → 측정 시작 후 **메인 창을 트레이로 숨긴 채** 핫키/오버레이로 종료 → ①"GitHub 백업 완료" 시스템 알림 ②백업 레포에 새 커밋(diff 확인) ③설정 "마지막 백업" 시각 갱신. 최초 백업 시 Windows 알림 권한 프롬프트가 뜨면 허용.
- **미설정인 경우**: 백업 설정을 비운 채 종료 → 알림·에러 없이 조용히(세션 저장 toast만).
- **연속 종료**: 짧은 세션을 연달아 종료해도 백업이 겹치지 않고 앱 정상.
- **오프라인/토큰 오류**: 세션 저장은 정상, 백업 실패는 알림·toast 없이 콘솔 에러만. 수동 백업은 설정 화면에서 여전히 가능.

## 범위 밖 (Not V4)
- 주기적/스케줄 백업(현재는 세션 종료 트리거만).
- 복구·수동 복원 경로에서의 자동 백업.
- 백업 실패 재시도 큐(현재는 실패 시 다음 세션 종료 때 다시 시도).
